## 文档编写规范

### 标题规范
- 一级标题：文档主标题（# 标题）
- 二级标题：主要章节（## 标题）
- 三级标题：子章节（### 标题）
- 四级标题：具体问题或要点（#### 标题）

### 格式规范

1. **代码块**
   - 必须指定语言类型
   - 添加必要的注释
   - 保持缩进一致

2. **列表**
   - 使用 `-` 作为无序列表标记
   - 使用数字作为有序列表标记
   - 保持层级清晰

3. **强调**
   - 使用 `**粗体**` 强调重点
   - 使用 `代码` 标记代码片段
   - 使用引用块标记重要提示

4. **链接**
   - 内部链接使用相对路径
   - 外部链接使用完整 URL
   - 链接文字要有意义

### 内容规范

1. **准确性**
   - 内容必须准确无误
   - 代码示例必须可运行
   - 及时更新过时内容

2. **完整性**
   - 包含必要的背景知识
   - 提供完整的代码示例
   - 添加相关文档链接

3. **可读性**
   - 使用简洁明了的语言
   - 合理使用图表和示例
   - 保持逻辑清晰

4. **实用性**
   - 关注实际应用场景
   - 提供最佳实践建议
   - 指出常见陷阱

## 特殊组件使用

### 提示框

```markdown
::: tip 提示
这是一个提示
:::

::: warning 警告
这是一个警告
:::

::: danger 危险
这是一个危险提示
:::

::: info 信息
这是一个信息提示
:::
```

### 代码组

```markdown
::: code-group
\`\`\`js [JavaScript]
console.log('Hello')
\`\`\`

\`\`\`ts [TypeScript]
console.log('Hello' as string)
\`\`\`
:::
```

### 自定义容器

```markdown
::: details 点击查看详情
详细内容...
:::
```

## 文件命名规范

- README.md: 技术栈主文档
- interview-bank.md: 面试题库
- [tech]-cheatsheet.md: 速查表
- [tech]-interview.md: 面试专题
- sources.md: 参考资料来源
- index.md: 目录索引

## 更新维护

- 每次更新需在文档头部标注更新时间
- 重大更新需在 AGENTS.md 中记录
- 定期检查外部链接有效性
- 及时补充最新的技术变化
